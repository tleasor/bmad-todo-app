## What Went Well

- **Adversarial code review process was effective.** Every review round caught real correctness bugs — not nitpicks. Story 1.1's typecheck gate was a no-op masking real TS errors; Story 1.3's migration runner had no duplicate-version pre-flight check. The review process found these before they shipped.
- **Architectural decisions held.** Monorepo structure, dependency caps (≤25 per package), and architectural boundaries scaled through all 11 stories without a major rethink.
- **Journey 1 and Journey 4 delivered as planned.** The user-facing surface is coherent: type a task, see it appear optimistically, handle network failures gracefully, deploy with `docker compose up`.
- **Deferred work was documented, not silently accumulated.** Every code review explicitly separated "patch now" from "defer with documentation." This is meaningful — it's documented debt, not hidden debt.

---

## Challenges

- **Bun test runtime has no DOM.** Hit in Stories 1.5, 1.6, 1.7. Forced added devDeps (bun-plugin-solid, babel-preset-solid, @babel/core, @babel/preset-typescript) and resulted in source-string grep tests instead of behavioral rendering tests.
- **Shared E2E dev SQLite DB.** Stories 1.8, 1.9, 1.10, 1.11 all deferred the same root cause — Playwright `fullyParallel: true` against a DB that accumulates rows. Story 1.11 patched settle helpers to work around it rather than fixing the root cause.
- **Defer decisions lacked triage.** Every deferral landed in `deferred-work.md` without a priority signal. At retrospective time, 60+ items look equally optional. Future reviews should label each defer: *pre-next-epic*, *resolved-by-story-X*, or *genuinely-optional*.
- **Docker runtime image is 447 MB** vs. the ~80 MB architecture baseline. devDependencies were not excluded from the runtime stage.
- **Eden type cast in `api.ts`.** `(api.api as unknown as ...)` is a compile-time lie — backend shape changes break at runtime, not compile time.
- **`useCreateTask` shared mutation observer race.** Per-row `retry()` closures share observer state; concurrent mutations race. Epic 2 adds more mutation types, which increases the concurrency surface.
- **`LiveRegion announce()` is fragile.** Module-scope signal, no queue, no mount sentinel. Rapid calls collapse; pre-mount calls are silently dropped. First real production use is in Epic 2 (toggle announcements).
- **`<h1>` reads 'bmad-todo-app'** — the repo identifier, not a product name. Flagged in Story 1.5, not addressed.

---

## Key Insights

1. **The review process works — the triage of deferred items needs improvement.** Catching bugs is good; leaving 60+ deferred items without priority signals creates the next retro's problem.
2. **Test infrastructure debt compounds fast.** The Bun DOM limitation and shared E2E DB were tolerable for one story; by Story 11 they required workarounds in every new story. Fix before Epic 2 adds more mutation surface.
3. **Foundation decisions were sound.** Being opinionated upfront (monorepo, dep caps, boundaries) paid off — no architectural rethink across 11 stories.
4. **Deferred-work triage at deferral time, not retrospective time.** Future reviews should categorize deferrals when they are written, not at the retro.
