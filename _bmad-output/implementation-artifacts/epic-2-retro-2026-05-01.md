## Challenges

1. **Four Epic 1 action items were never done.** No tracking slot, no named deadline, no one noticed. Process gap.
2. **Eden type cast extended rather than resolved.** We added a callable-interface extension on top of the existing cast rather than finding a clean solution. The workaround works; the root problem persists.
3. **Concurrent mutation races deferred.** Rapid-click toggle fires two PATCH requests with conflicting `completed` states; timer/sync-store state races. Documented and deferred.
4. **`onSuccess` discards server response — intentional for toggle, risky for Epic 3.** Toggle's optimistic-final policy (no rollback, no cache update from server) works cleanly for toggle. Delete/undo requires pre-deletion snapshot for restore-at-original-position; this pattern doesn't transfer.
5. **`aria-label` static in both checkbox states.** "Mark task as complete" shown even when task is already complete. Caught in review; should be caught earlier during UX review of interactive states.

---

## Key Insights

1. **Prep items without a tracking slot don't get done.** "Operational clean-up before Epic X starts" is not a real deadline unless it lives in sprint-status or a named story task.
2. **Epic 3's UndoSnackbar is genuinely new territory.** No analog in the codebase — timed overlay, concurrent-delete collapsing, restore-at-original-position, Cmd/Ctrl+Z with platform detection. Needs explicit architectural decision before Story 3.4.
3. **The optimistic-final cache policy needs revisiting for delete.** `onSuccess` discarding the server response works when the mutation is reversible. Delete is irreversible without the undo snapshot.
4. **Review Decision items (not just Patch/Defer) signal healthy architectural surface area.** Story 2.2 had two explicit design decisions surfaced in review. That's the process working, not failing.
